# AI-Ready VM Repository Template

![CI](https://github.com/caviri/ai-ready-vm-repository-template/actions/workflows/ci.yml/badge.svg)

A repository template providing a reproducible Ubuntu-based dev environment with
pre-installed AI coding agents, SSH access, credential persistence across rebuilds,
and per-project port configuration via Docker Compose.

## Features

- **Dev container**: Ubuntu 22.04, Python 3.12, Node 20
- **AI coding agents**: Claude Code, Codex, Pi, Gemini CLI — installed and ready
- **VS Code extensions**: Official extensions for all four agents, plus GitLens, Docker, EditorConfig
- **SSH server**: Password and key-based auth, configurable port
- **Credential persistence**: Bind mounts survive container rebuilds
- **Per-project ports**: No conflicts when running multiple containers on the same host
- **Auto-changelog**: git-cliff generates `CHANGELOG.md` on every commit
- **Conventional Commits**: Enforced via commit-msg hook
- **Pre-commit hooks**: Trailing whitespace, YAML validation, large file checks
- **CI**: GitHub Actions with `just check`
- **Repo hygiene**: Issue templates, PR template, CODEOWNERS, Dependabot, EditorConfig

## Quick start

### Option A: Local Docker + VS Code

1. Install [Docker](https://docs.docker.com/get-docker/) and [VS Code](https://code.visualstudio.com/)
2. Install the [Dev Containers](https://marketplace.visualstudio.com/items?itemName=ms-vscode-remote.remote-containers) extension
3. Clone this repo and open it in VS Code
4. Copy `.env.example` to `.env` and fill in your API keys and ports
5. Run **Dev Containers: Reopen in Container** from the command palette

### Option B: Remote Ubuntu host via SSH

1. SSH into your Ubuntu host
2. Ensure Docker is installed
3. Clone the repo on the host
4. Open the host in VS Code with [Remote - SSH](https://marketplace.visualstudio.com/items?itemName=ms-vscode-remote.remote-ssh)
5. Copy `.env.example` to `.env` and configure
6. Reopen in the dev container

## Port configuration

Each project on the same host needs unique ports. Edit `.env`:

```bash
# Project A
SSH_PORT=2222
DEV_PORT=8888

# Project B (different repo, different .env)
SSH_PORT=2223
DEV_PORT=8889
```

Defaults are `2222` and `8888` if `.env` is absent.

## SSH access

### Password authentication

```bash
ssh vscode@<host> -p 2222
# Password: vscode
```

### Key-based authentication

Drop your public key into `.devcontainer/persist/ssh/authorized_keys`.
It persists across container rebuilds.

### After container rebuild

Host keys change on rebuild. Clear the old key on your client:

```bash
ssh-keygen -R "[<host>]:2222"
```

## AI coding agents

All agents are pre-installed in the container and have VS Code extensions configured:

| Agent | CLI command | VS Code extension | Auth |
|-------|------------|-------------------|------|
| Claude Code | `claude` | Anthropic.claude-code | `ANTHROPIC_API_KEY` or browser login |
| Codex | `codex` | OpenAI.chatgpt | `OPENAI_API_KEY` or browser login |
| Pi | `pi` | — | Per-provider config |
| Gemini CLI | `gemini` | Google.gemini-cli-vscode-ide-companion | `GOOGLE_API_KEY` or browser login |

API keys are loaded from `.env` into the container via Docker Compose `env_file`.
Credentials and configs persist in `.devcontainer/persist/` across rebuilds.

## Credential persistence

Five directories are bind-mounted from the host into the container:

| Host path | Container path | Purpose |
|-----------|---------------|---------|
| `persist/config/` | `~/.config` | CLI configs (Claude, Gemini, etc.) |
| `persist/local-share/` | `~/.local/share` | Tool data |
| `persist/cache/` | `~/.cache` | pip, npm, tool caches |
| `persist/pi/` | `~/.pi` | Pi agent config and extensions |
| `persist/ssh/` | `~/.ssh` | SSH keys and authorized_keys |

Contents are gitignored. Directories are tracked via `.gitkeep`.

## Conventional Commits and changelog

All commits must follow [Conventional Commits](https://www.conventionalcommits.org/):

```
feat: add user authentication
fix(api): handle timeout errors
docs: update SSH instructions
```

The `commit-msg` hook rejects non-conforming messages.
After each commit, `CHANGELOG.md` is automatically regenerated by
[git-cliff](https://git-cliff.org/) via the `post-commit` hook.

## Project structure

```
.devcontainer/          Dev container (Dockerfile, Compose, hooks)
  persist/              Credential/config storage (gitignored contents)
.github/                CI, issue templates, PR template, CODEOWNERS, Dependabot
src/                    Source code
tests/                  Tests
docs/                   Documentation
scripts/                Helper scripts
justfile                Command runner (just recipes)
cliff.toml              git-cliff changelog config
.pre-commit-config.yaml Pre-commit hook config
CLAUDE.md               Project context for Claude Code
AGENTS.md               Project context for Codex
```

## Common commands

```bash
just                # List all available commands
just setup          # Run project setup
just check          # Run all pre-commit checks
just pre-commit     # Same as check (callable from hooks)
just post-commit    # Regenerate changelog and amend commit
just changelog      # Regenerate changelog (no amend)
just test           # Run tests (placeholder)
just format         # Run formatter (placeholder)
just clean          # Remove temp/cache files
just ssh-status     # Check SSH server status and port
```
